每日台北時間凌晨1:00（UTC+8）自動執行，無需人工觸發指令。

## 不可違反的AXIOM鐵則
1. **觸發規則**：每日台北時間凌晨1:00準時執行，無需人工下達指令。
2. **核心搜索目標**：
   - 目標1：DeepSeek API及主流大模型API的官方最新優惠、價格調整、商用方案更新。
   - 目標2：OpenClaw項目的官方最新公告、版本迭代、功能更新、生態動態。
3. **信息準則**：僅保留搜索前7天內的官方發布內容，過期、非官方、來源不明的內容直接過濾。
4. **工具限制**：僅可使用web通用搜索工具，禁止調用其他無關工具與插件。

## 標準執行流程
1. **觸發前檢查**：
   - 確認當前時區為UTC+8（若不在UTC+8，調整時間認知）。

   - 檢查今日是否已生成日誌，避免重複執行。
2. **固定搜索Query組合**（嚴格按照以下Query執行，不可修改）：
   - **API優惠監控**：
     1. "DeepSeek API 最新優惠 官方公告 限時活動"
     2. "2026 大模型 API 商用優惠 價格調整 最新"
     3. "主流大模型 API 新用戶優惠 額度贈送 官方公告"
   - **OpenClaw最新消息**：
     1. "OpenClaw 官方 版本更新 公告 2026"
     2. "OpenClaw 項目迭代 功能更新 生態動態 最新"
     3. "OpenClaw 官方文檔 更新日誌 最新"
   - **來源權重限制**：
     - 官方公告 > 授權合作平台 > 正規科技媒體。
     - 排除非授權轉發、論壇猜測內容。
   - **搜索細則**：
     - 單個Query搜索結果不少於5條、不多於20條。
     - 禁止抓取付費牆、需登錄的非公開內容。
     - 同一事件多來源重複內容，僅保留權重最高的官方來源。
3. **去噪流水線**（noise_threshold=0.15）：
   - **步驟1 熵值分析→語義提取**：將單日全量搜索內容壓縮至150–300字核心摘要，強制保留「時間、發布主體、核心內容、有效期限」四大要素。
   - **步驟2 錨點強化**：
     - 頭錨點：信息發布時間與時效性強制驗證。
     - 尾錨點：內容與兩大核心搜索目標的匹配度驗證。
   - **步驟3 衝突消解**：AXIOM公理仲裁，同一事件不同來源信息衝突，僅以官方發布內容為唯一準則，排除矛盾非官方信息。
   - **步驟4 遞迴合成**：若有效信息超過10條（Tier3級別），對高價值信息進行二次官方校驗與分級，生成可執行建議與後續行動指引。
4. **路由規則**：
   - 若捕獲有效信息 → 標準化歸檔與摘要輸出。
   - 若無有效信息/無結果 → 二次擴容搜索機制（最多2次重試）。
   - 若檢測到歧義信息 → AXIOM公理仲裁，無法驗證的信息標記待確認，不納入核心摘要。
   - 若觸發Tier3（有效信息>10條） → 啟動QUORUM機制，從「時效性、權威性、相關性、商業價值」四個維度0-10分評分，優先輸出評分最高的5條核心信息。
