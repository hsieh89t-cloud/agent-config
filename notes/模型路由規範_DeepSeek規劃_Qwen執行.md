# 模型路由規範（DeepSeek 規劃 / Qwen3.5 執行）

## 目標
在有限資源下，將高推理成本任務集中在雲端規劃層（DeepSeek），本地模型（Qwen3.5:9b）負責可模板化、可批次化的執行層工作，提升穩定性與成本效率。

---

## 一、角色分工

### A. DeepSeek（規劃層）
適用任務：
1. 任務拆解與優先序排序
2. SOP / 流程圖 / 路由規則設計
3. 法律推理框架、爭點清單、風險盤點
4. 產出「執行指令包」（給 Qwen3.5）

輸出要求：
- 僅產生結構化規劃，不直接產最終長文
- 每次輸出必含：
  - `Task Goal`
  - `Input Schema`
  - `Processing Steps`
  - `Output Schema`
  - `QC Checklist`

### B. Qwen3.5:9b（執行層，本地）
適用任務：
1. 依規劃批次產文（摘要、白話化、重寫）
2. 模板填充（Obsidian 筆記欄位、固定格式文件）
3. 前處理（切段、清洗、標註、命名）
4. 目錄整理與檔案標準化

輸出要求：
- 嚴格遵守模板與欄位
- 不新增未授權欄位
- 遇到不確定法條或引用時標記 `NEEDS_VERIFICATION`

---

## 二、路由判斷規則

### 規則（先判斷再執行）
- 問題是「**怎麼做**」→ DeepSeek
- 問題是「**照規則做出來**」→ Qwen3.5
- 需要多步推理、策略取捨、風險評估 → DeepSeek
- 大量重複、格式化、批次加工 → Qwen3.5

### 法律任務加嚴規則
- 法律解釋策略 / 攻防論證架構：DeepSeek
- 法條逐條轉寫、摘要、索引整理：Qwen3.5
- 任何引用無法驗證：停止輸出結論，回報 `NEEDS_VERIFICATION`

---

## 三、每日批次節奏（建議）

- 每日總量：5–6 條
- 批次流程：
  1. DeepSeek 先做一次「當日批次規劃」（5–10 分鐘）
  2. 輸出 5–6 份執行指令包
  3. 全部交由 Qwen3.5 本地執行
  4. 最後做一次 QC 彙總

---

## 四、提示詞模板

### 1) DeepSeek 規劃模板
```
你是規劃引擎。請把任務拆成可由本地模型執行的指令包。
輸出格式固定為：
1) Task Goal
2) Input Schema
3) Processing Steps（最多 7 步）
4) Output Schema
5) QC Checklist（可驗證條件）
限制：不產最終長文，只產執行計畫。
```

### 2) Qwen3.5 執行模板
```
你是執行引擎。依照下列指令包產出，不要改欄位名稱。
若缺資料或引用無法驗證，輸出 NEEDS_VERIFICATION 與缺口清單。
禁止捏造法條或來源。
```

---

## 五、失敗降級策略（低資源模式）

1. Qwen3.5 記憶體壓力過高時：
   - 降低單批輸入長度
   - 降低同時任務數（建議 1–2）
   - 拆批（一份文件拆成多段）

2. DeepSeek 暫時不可用時：
   - 使用既有 SOP 模板直接給 Qwen3.5 跑
   - 規劃層改為人工確認後再執行

3. 任何模型錯誤時：
   - 保留中間產物
   - 記錄失敗原因與當前批次 ID
   - 下一輪從中斷點續跑

---

## 六、最小可用操作規範（MVP）

- 先規劃後執行，不跳步
- 每批次都要有 QC 清單
- 不可驗證引用不得輸出為確定結論
- 每日控制 5–6 條，優先穩定完成率

---

## 七、短結論

這套路由的核心是：
- **DeepSeek 負責思考與規劃**（少量、高價值）
- **Qwen3.5 負責執行與批次產出**（大量、低成本）

在目前資源條件下，這是穩定性與成本最平衡的做法。
