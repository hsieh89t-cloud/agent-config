## structured_memory_framework
*基於 AgentMemory 和 Adaptive Learning Agent 設計理念優化的結構化記憶與學習框架*

### 自適應學習系統 (Adaptive Learning System)
*基於 Adaptive Learning Agent 設計理念*

#### 學習記錄格式
- **成功學習**：`[學習ID] 內容 | 類別 | 來源 | 日期 | 上下文`
- **錯誤記錄**：`[錯誤ID] 描述 | 情境 | 解決方案 | 預防提示 | 狀態`
- **範例**：
  - `[L001] 使用 claude-sonnet 處理 90% 任務，更快更便宜 | technique | successful-pattern | 2026-03-08 | 模型選擇`
  - `[E001] JSON 解析在 null 值時失敗 | API 回應處理 | 添加 null 檢查 | 始終驗證輸入 | resolved`

#### 學習類別系統 (Learning Categories)
- **technique**：技術方法、策略、工作流程
- **bug-fix**：錯誤修復與解決方案
- **best-practice**：最佳實踐與標準
- **constraint**：限制條件與邊界
- **api-quirk**：API 特性與注意事項
- **error-handling**：錯誤處理模式
- **optimization**：效能優化技巧
- **security**：安全考量與防護

#### 學習來源追蹤 (Learning Sources)
- **user-correction**：使用者修正與指導
- **error-discovery**：錯誤發現與解決
- **successful-pattern**：成功模式識別
- **user-feedback**：使用者回饋與建議
- **system-observation**：系統觀察與分析
- **experimentation**：實驗與測試結果

### 事實記憶 (Fact Memory)
- **格式**：`[日期] 事實描述 | 標籤1,標籤2 | 來源 | 可信度`
- **範例**：`[2026-03-08] 安裝 Academic Research Skill，使用 OpenAlex API 搜尋學術論文 | skills,academic,api | 手動安裝 | high`
- **查詢**：可使用 `grep "標籤" MEMORY.md` 或文字搜尋

### 經驗學習 (Experience Learning)
- **成功模式**：`情境 → 行動 → 結果 → 洞察 → 可應用性`
- **失敗模式**：`情境 → 行動 → 問題 → 根本原因 → 改進方案 → 預防措施`
- **範例**：`法律任務 → 採用哨兵+QC雙層防線 → 避免幻覺法條 → 對法律內容必須經雙層驗證 → 所有法律相關任務`

### 實體追蹤 (Entity Tracking)
- **人物**：`名稱 | 角色 | 關聯專案 | 最後互動 | 關係強度`
- **專案**：`名稱 | 狀態 | 進度 | 負責人 | 關鍵里程碑 | 風險等級`
- **技能**：`名稱 | 類型 | 風險等級 | 安裝日期 | 使用頻率 | 效能評分`

### 學習統計與追蹤
```markdown
## 學習統計 (Learning Statistics)
- **總學習記錄**：X
- **成功學習數**：Y (technique: A, best-practice: B, ...)
- **錯誤記錄數**：Z (resolved: R, unresolved: U)
- **學習來源分佈**：user-correction: P%, error-discovery: Q%, ...
- **最近學習**：最近 7 天新增 N 個學習
- **學習應用率**：M% 的學習被後續任務引用
```

### 記憶查詢與檢索系統
```bash
# 搜尋特定類別的學習
grep -n "類別.*technique" MEMORY.md

# 搜尋錯誤解決方案
grep -n "錯誤ID.*E[0-9]" MEMORY.md

# 取得學習摘要
sed -n '/## 學習統計/,/## /p' MEMORY.md

# 搜尋未解決錯誤
grep -n "狀態.*unresolved" MEMORY.md

# 按來源搜尋學習
grep -n "來源.*user-correction" MEMORY.md
```

### 學習維護與應用流程
1. **即時記錄**：在錯誤發生或學習產生時立即記錄
2. **每日整理**：整理當日學習，歸類與標籤化
3. **每週回顧**：
   - 檢視本週學習統計
   - 識別重複錯誤模式
   - 提煉高價值學習為最佳實踐
4. **每月優化**：
   - 評估學習系統效能
   - 優化分類與查詢機制
   - 清理低價值或過時學習
5. **任務前檢索**：在開始新任務前，檢索相關學習與錯誤預防

### 錯誤預防與品質保證
1. **錯誤模式分析**：識別常見錯誤模式與根本原因
2. **檢查點系統**：在關鍵步驟前加入檢查點
3. **預防措施**：針對高頻錯誤建立標準預防流程
4. **品質指標**：追蹤錯誤率、解決時間、重複錯誤率

- **Obsidian 記憶中樞**：作為學習的可回滾歸檔與分析
